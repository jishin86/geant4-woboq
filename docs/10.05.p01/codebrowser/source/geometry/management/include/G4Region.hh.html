78// The type G4RegionManager is introduced to encapsulate the methods used by
79// both the master thread and worker threads to allocate memory space for
80// the fields encapsulated by the class G4RegionData. When each thread
81// initializes the value for these fields, it refers to them using a macro
82// definition defined below. For every G4Region instance, there is a
83// corresponding G4RegionData instance. All G4RegionData instances are
84// organized by the class G4RegionManager as an array.
85// The field "int instanceID" is added to the class G4Region.
86// The value of this field in each G4Region instance is the subscript
87// of the corresponding G4RegionData instance.
88// In order to use the class G4RegionManager, we add a static member in
89// the class G4Region as follows: "static G4RegionManager subInstanceManager".
90// For the master thread, the array for G4RegionData instances grows
91// dynamically along with G4Region instances are created. For each worker
92// thread, it copies the array of G4RegionData instances from the master thread.
93// In addition, it invokes a method similiar to the constructor explicitly
94// to achieve the partial effect for each instance in the array.
95//
96typedef G4GeomSplitter<G4RegionData> G4RegionManager;
