29// class G4LogicalVolume
30//
31// Class description:
32//
33// Represents a leaf node or unpositioned subtree in the geometry hierarchy.
34// Logical volumes are named, and may have daughters ascribed to them.
35// They are responsible for retrieval of the physical and tracking attributes
36// of the physical volume that it represents: solid, material, magnetic field,
37// and optionally, user limits, sensitive detectors, regions, biasing weights.
38//
39// Get and Set functionality is provided for all attributes, but note that
40// most set functions should not be used when the geometry is `closed'.
41// As a further development, `Guard' checks can be added to ensure
42// only legal operations at tracking time.
43//
44// On construction, solid, material and name must be specified.
45//
46// Daughters are ascribed and managed by means of a simple
47// GetNoDaughters,Get/SetDaughter(n),AddDaughter interface.
48//
49// Smart voxels as used for tracking optimisation. They're also an attribute.
50//
51// Logical volumes self register to the logical volume Store on construction,
52// and deregister on destruction.
53//
54// NOTE: This class is currently *NOT* subclassed, since not meant to
55// act as a base class. Therefore, the destructor is NOT virtual.
56//
57// Data members:
58//
59// std::vector<G4VPhysicalVolume*> fDaughters
60// - Vector of daughters. Given initial size of 0.
61// G4FieldManager* fFieldManager
62// - Pointer (possibly 0) to (magnetic or other) field manager object.
63// G4Material* fMaterial
64// - Pointer to material at this node.
65// G4String fName
66// - Name of logical volume.
67// G4VSensitiveDetector *fSensitiveDetector
68// - Pointer (possibly 0) to `Hit' object.
69// G4VSolid* fSolid
70// - Pointer to solid.
71// G4UserLimits* fUserLimits
72// - Pointer (possibly 0) to user Step limit object for this node.
73// G4SmartVoxelHeader* fVoxel
74// - Pointer (possibly 0) to optimisation info objects.
75// G4bool fOptimise
76// - Flag to identify if optimisation should be applied or not.
77// G4bool fRootRegion
78// - Flag to identify if the logical volume is a root region.
79// G4double fSmartless
80// - Quality for optimisation, average number of voxels to be spent
81// per content.
82// const G4VisAttributes* fVisAttributes
83// - Pointer (possibly 0) to visualization attributes.
84// G4Region* fRegion
85// - Pointer to the cuts region (if any)
86// G4MaterialCutsCouple* fCutsCouple
87// - Pointer (possibly 0) to associated production cuts.
88// G4double fBiasWeight
89// - Weight used in the event biasing technique.
90//
91// Following data members has been moved to G4Region - M.Asai (Aug/18/2005)
92// G4FastSimulationManager* fFastSimulationManager
93// - Pointer (possibly 0) to G4FastSimulationManager object.
94// G4bool fIsEnvelope
95// - Flags if the Logical Volume is an envelope for a FastSimulationManager.

169// The type G4LVManager is introduced to encapsulate the methods used by
170// both the master thread and worker threads to allocate memory space for
171// the fields encapsulated by the class G4LVData. When each thread
172// initializes the value for these fields, it refers to them using a macro
173// definition defined below. For every G4LogicalVolume instance, there is
174// a corresponding G4LVData instance. All G4LVData instances are organized
175// by the class G4LVManager as an array.
176// The field "int instanceID" is added to the class G4LogicalVolume.
177// The value of this field in each G4LogicalVolume instance is the subscript
178// of the corresponding G4LVData instance.
179// In order to use the class G4LVManager, we add a static member in the class
180// G4LogicalVolume as follows: "static G4LVManager subInstanceManager".
181// For the master thread, the array for G4LVData instances grows dynamically
182// along with G4LogicalVolume instances are created. For each worker thread,
183// it copies the array of G4LVData instances from the master thread.
184// In addition, it invokes a method similiar to the constructor explicitly
185// to achieve the partial effect for each instance in the array.
186//
187typedef G4GeomSplitter<G4LVData> G4LVManager;

257 // Returns the mass of the logical volume tree computed from the
258 // estimated geometrical volume of each solid and material associated
259 // to the logical volume and (by default) to its daughters.
260 // NOTE: the computation may require a considerable amount of time,
261 // depending from the complexity of the geometry tree.
262 // The returned value is cached and can be used for successive
263 // calls (default), unless recomputation is forced by providing
264 // 'true' for the boolean argument in input. Computation should
265 // be forced if the geometry setup has changed after the previous
266 // call. By setting the 'propagate' boolean flag to 'false' the
267 // method returns the mass of the present logical volume only
268 // (subtracted for the volume occupied by the daughter volumes).
269 // An optional argument to specify a material is also provided.
270 void ResetMass();
271 // Ensure that cached value of Mass is invalidated - due to change in
