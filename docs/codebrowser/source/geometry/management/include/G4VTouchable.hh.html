29// class G4VTouchable
30//
31// Class description:
32//
33// Base class for `touchable' objects capable of maintaining an
34// association between parts of the geometrical hierarchy (volumes
35// &/or solids) and their resultant transformation.
36//
37// Utilisation:
38// -----------
39// A touchable is a geometrical volume (solid) which has a unique
40// placement in a detector description. It is an abstract base class which
41// can be implemented in a variety of ways. Each way must provide the
42// capabilities of obtaining the transformation and solid that is described
43// by the touchable.
44//
45// All G4VTouchable implementations must respond to the two following
46// "requests":
47//
48// 1) GetTranslation and GetRotation that return the components of the
49// volume's transformation.
50//
51// 2) GetSolid that gives the solid of this touchable.
52//
53//
54// Additional capabilities are available from implementations with more
55// information. These have a default implementation that causes an exception.
56//
57// Several capabilities are available from touchables with physical volumes:
58//
59// 3) GetVolume gives the physical volume.
60//
61// 4) GetReplicaNumber or GetCopyNumber gives the copy number of the
62// physical volume, either if it is replicated or not.
63//
64// Touchables that store volume hierarchy (history) have the whole stack of
65// parent volumes available. Thus it is possible to add a little more state
66// in order to extend its functionality. We add a "pointer" to a level and a
67// member function to move the level in this stack. Then calling the above
68// member functions for another level, the information for that level can be
69// retrieved.
70//
71// The top of the history tree is, by convention, the world volume.
72//
73// 5) GetHistoryDepth gives the depth of the history tree.
74//
75// 6) GetReplicaNumber/GetCopyNumber, GetVolume, GetTranslation and
76// GetRotation each can be called with a depth argument.
77// They return the value of the respective level of the touchable.
78//
79// 7) MoveUpHistory(num) moves the current pointer inside the touchable
80// to point "num" levels up the history tree. Thus, eg, calling
81// it with num=1 will cause the internal pointer to move to the mother
82// of the current volume.
83// NOTE: this method MODIFIES the touchable.
84//
85// An update method, with different arguments is available, so that the
86// information in a touchable can be updated:
87//
88// 8) UpdateYourself takes a physical volume pointer and can additionally
89// take a NavigationHistory.
90
91// History:
92// Created: Paul Kent, August 1996
93// --------------------------------------------------------------------
