109/** Andrea Dotti Feb 2015
110 * GPS messenger design requires some explanation for what distributions
111 * parameters are concerned : Each thread has its own GPS
112 * since primary generation is a user action.
113 * However to save memory the underlying structures that provide the
114 * GPS functionalities ( the G4SPS*Distribution classes and the
115 * G4SPSRandomGenerator class)
116 * are shared among threads. This implies that modifying parameters of sources
117 * requires some attention:
118 * 1- Only one thread should change source parameters.
119 * 2- Changing of parameters can happen only between runs, when is guaranteed
120 * that no thread is accessing them
121 * 2- UI commands require that even if messenger is instantiated in a thread
122 * the commands are executed in the master (this is possible since V10.1)
123 * The simplest solution is to use UI commands to change GPS parameters and
124 * avoid C++ APIs. If this is inevitable a simple solution is to instantiate
125 * an instance of G4GeneralParticleSource explicitly in the master thread
126 * (for example in G4VUserActionInitialization::BuildForMaster() and set the
127 * defaults parameter there).
128 */
