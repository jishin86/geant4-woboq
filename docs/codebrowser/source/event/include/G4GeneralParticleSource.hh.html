159/** Andrea Dotti Feb 2015
160 * GPS messenger design requires some explanation for what distributions
161 * parameters are concerned : Each thread has its own GPS
162 * since primary generation is a user action.
163 * However to save memory the underlying structures that provide the
164 * GPS functionalities ( the G4SPS*Distribution classes and the
165 * G4SPSRandomGenerator class)
166 * are shared among threads. This implies that modifying parameters of sources
167 * requires some attention:
168 * 1- Only one thread should change source parameters.
169 * 2- Changing of parameters can happen only between runs, when is guaranteed
170 * that no thread is accessing them
171 * 2- UI commands require that even if messenger is instantiated in a thread
172 * the commands are executed in the master (this is possible since V10.1)
173 * The simplest solution is to use UI commands to change GPS parameters and
174 * avoid C++ APIs. If this is inevitable a simple solution is to instantiate
175 * an instance of G4GeneralParticleSource explicitly in the master thread
176 * (for example in G4VUserActionInitialization::BuildForMaster() and set the
177 * defaults parameter there).
178 */
